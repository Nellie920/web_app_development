# 產品需求文件 (PRD)

## 1. 專案概述

- **專案名稱**：個人記帳簿
- **背景與動機**：為了了解日常開銷與投資狀況，需要一個可以整合生活收支與股票損益的工具，幫助自己隨時掌握財務狀況。此外，希望透過更具風格的系統介面來降低記帳帶來的枯燥感。
- **目標用戶**：我自己
- **核心價值主張**：提供一個介面圓潤可愛、操作直覺，且兼具日常記帳與基礎股票損益追蹤功能的個人財務管理輔助工具。

## 2. 功能需求

以下為系統核心功能的使用者故事（User Story）：

1. **計算收入及支出並顯示餘額**
   - 作為 **使用者**，我希望 **能夠記錄每筆收入與支出，並讓系統自動計算顯示當前餘額**，以便 **隨時掌握目前的財務狀況是否透支**。

2. **進行收入及支出種類分類**
   - 作為 **使用者**，我希望 **能在記帳時設定不同的種類標籤（例如：飲食、交通、薪水等）**，以便 **後續能清楚檢視各類別的開銷分佈**。

3. **介面圓潤可愛**
   - 作為 **使用者**，我希望 **應用程式的介面採用圓角設計、柔和色彩與可愛的視覺互動風格**，以便 **在日常記帳時能感到輕鬆愉悅，提升持續記帳的動力**。

4. **可計算股票損益**
   - 作為 **使用者**，我希望 **能輸入股票的買賣紀錄及金額**，以便 **系統自動為我計算投資的整體損益情況，統整於個人財務中**。

5. **可支援校正功能**
   - 作為 **使用者**，我希望 **能夠隨時修改或刪除過去輸入錯誤的記帳紀錄或股票紀錄**，以便 **確保所有的財務數據皆保持精確與真實**。

## 3. 非功能需求

- **技術限制**：
  - 後端架構：使用 Python 語言與 Flask 框架。
  - 頁面渲染：使用 Flask 內建的 Jinja2 模板引擎。
  - 資料儲存：使用 SQLite 輕量級關聯式資料庫。
- **效能與安全考量**：
  - 介面設計須確保在加入圓潤可愛的樣式（如 CSS 動畫）時，不會造成頁面載入過於緩慢或卡頓。
  - 針對使用者輸入的金額與文字，需有基本的防護與驗證機制（防範跨站腳本攻擊 XSS 或 SQL Injection）。

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須有)**：
  - 基礎收支紀錄的新增、修改、刪除（CRUD 校正功能）。
  - 自動計算包含總收入、總支出與當下可用餘額。
  - 收支種類分類紀錄功能。
- **Should Have (應該有)**：
  - 基本股票紀錄的輸入與簡易損益計算功能。
  - 基礎版「圓潤可愛」設計的主題 CSS，包含圓角按鈕與溫暖色系。
- **Nice to Have (加分項)**：
  - 以圓餅圖或長條圖顯示分類花費（進階報表）。
  - 更豐富流暢的動態介面效果或自訂主題。
